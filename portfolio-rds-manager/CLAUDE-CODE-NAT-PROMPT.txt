# NAT GATEWAY MANAGER - CLAUDE CODE SETUP PROMPT

Copy this prompt to Claude Code to set up the NAT Gateway Manager.

```
NAT GATEWAY MANAGER - SETUP & CONFIGURATION

I need help setting up the Portfolio NAT Gateway Manager to manage my existing NAT Gateway.

PREREQUISITES:
- AWS CLI is configured with valid credentials
- I have existing NAT Gateway(s) running in AWS
- All NAT Manager files are in this directory
- I want to manage NAT Gateway lifecycle (stop/start) for cost savings

WORKFLOW:

═══════════════════════════════════════════════════════════════════════════════

PHASE 1: DISCOVER EXISTING NAT GATEWAYS (5 minutes)

Step 1.1: List All NAT Gateways
  - Run: aws ec2 describe-nat-gateways --filter Name=state,Values=available
  - Show the full output in a table format
  - Extract and display:
    * NAT Gateway IDs
    * Subnet IDs where they run
    * Elastic IP addresses
    * Availability zones

Step 1.2: List Elastic IP Addresses
  - Run: aws ec2 describe-addresses --domain vpc
  - Show all EIPs (both in use and available)
  - Note which ones are associated with NAT Gateways

Step 1.3: List Route Tables
  - Run: aws ec2 describe-route-tables
  - Show route table IDs and their associated subnets
  - Identify which route tables have routes through NAT Gateways

Step 1.4: Get AWS Account ID
  - Run: aws sts get-caller-identity
  - Extract account ID

Step 1.5: Summarize Findings
  - List all NAT Gateways you found
  - List all Elastic IPs
  - Show which route tables use NAT Gateways
  - Ask me to confirm which NAT Gateways I want to manage

═══════════════════════════════════════════════════════════════════════════════

PHASE 2: CONFIGURE THE TOOL (5 minutes)

Step 2.1: Read Current Configuration
  - Read: nat-portfolio-config.yaml
  - Show the structure

Step 2.2: Update NAT Gateway Configuration
  For each NAT Gateway the user confirmed:
  - Update "subnet_id" with the actual subnet ID where NAT Gateway runs
  - Update "allocation_id" with the Elastic IP allocation ID
  - Update "route_tables" with the route table IDs that use this NAT
  - Update "availability_zone" with the correct AZ
  - Update the account ID from 311330778203 to the actual account ID
  - Keep route_destination as "0.0.0.0/0"
  - Update tags as needed

Step 2.3: Verify Configuration
  - Show the updated nat-portfolio-config.yaml
  - Ask user to confirm it's correct
  - If corrections needed, ask for them and update

Step 2.4: Save Configuration
  - Make scripts executable: chmod +x portfolio-nat-manager.sh
  - Confirm config file is saved

═══════════════════════════════════════════════════════════════════════════════

PHASE 3: INITIALIZE STATE (2 minutes)

Step 3.1: Create State File
  - Create .portfolio-nat-state.json with initial state
  - For each NAT Gateway, record:
    * nat_gateway_id (the current NAT Gateway ID)
    * elastic_ip (the allocation ID)
    * status (available)
    * creation_date (today's date)

Step 3.2: Verify State File
  - Show the contents of .portfolio-nat-state.json
  - Confirm all NAT Gateways are tracked

═══════════════════════════════════════════════════════════════════════════════

PHASE 4: TEST THE TOOL (3 minutes)

Step 4.1: Test List Command
  - Run: ./portfolio-nat-manager.sh list
  - Should show all configured NAT Gateways

Step 4.2: Test Status Command
  - Run: ./portfolio-nat-manager.sh status
  - Should show all NAT Gateways and their current costs

Step 4.3: Test Cost Estimate
  - Run: ./portfolio-nat-manager.sh cost-estimate
  - Should show costs and savings potential

Step 4.4: Verify All Tests Pass
  - If any test fails, show the error and suggest fixes

═══════════════════════════════════════════════════════════════════════════════

PHASE 5: FINAL REPORT (1 minute)

Step 5.1: Generate Setup Summary
  - Create a summary showing:

📊 NAT GATEWAY MANAGER SETUP COMPLETE ✅

Configured NAT Gateways:
  ├─ Name:               [NAT Gateway names]
  ├─ Availability Zone:  [AZ]
  ├─ Subnet ID:          [Subnet]
  ├─ Elastic IP:         [EIP address]
  ├─ Route Tables:       [Route table IDs]
  └─ Monthly Cost:       ~$40/month (running)

Account ID: [Your Account ID]
Region: us-east-1
Configuration File: nat-portfolio-config.yaml
State File: .portfolio-nat-state.json

═══════════════════════════════════════════════════════════════════════════════

🎯 NEXT STEPS:

1. Test Stop/Start Workflow
   To pause NAT Gateway and save money:
   ./portfolio-nat-manager.sh stop primary-nat
   
   You'll lose outbound internet access from private subnets.
   Cost drops to ~$0.32/month (just Elastic IP storage).

2. Recreate NAT Gateway
   When you need it again:
   ./portfolio-nat-manager.sh start primary-nat
   
   Takes 2-3 minutes to recreate.
   Cost resumes to ~$40/month.

3. Monitor Status
   Check anytime:
   ./portfolio-nat-manager.sh status

4. View Savings Potential
   See how much you could save:
   ./portfolio-nat-manager.sh cost-estimate

═══════════════════════════════════════════════════════════════════════════════

💡 IMPORTANT NOTES:

- NAT Gateways CANNOT be paused (only deleted)
- Elastic IP addresses are preserved and reused
- Route tables are updated automatically during stop/start
- Stopping = Deleting the NAT Gateway (save money)
- Starting = Creating a new NAT Gateway with same config
- Private subnets lose outbound internet when NAT is deleted
- Setup takes 2-3 minutes to restore

═══════════════════════════════════════════════════════════════════════════════

Ready? Begin with Phase 1.
```

## HOW TO USE THIS PROMPT

### Step 1: Copy the Prompt
Copy everything between the triple backticks above.

### Step 2: Run Claude Code
```bash
cd ~/Code/GitHub/portfolio-rds-manager  # Same directory as RDS manager
claude-code

# At the prompt (>), paste the entire NAT Gateway setup prompt
```

### Step 3: Follow Claude Code's Instructions
Claude Code will:
1. Find your existing NAT Gateways in AWS
2. Capture their details (subnet, EIP, route tables)
3. Update nat-portfolio-config.yaml automatically
4. Initialize the state file
5. Test the tool
6. Show you how to use it

### Step 4: Start Using
After setup completes:
```bash
./portfolio-nat-manager.sh status           # See current state
./portfolio-nat-manager.sh cost-estimate    # View savings
./portfolio-nat-manager.sh stop primary-nat # Delete NAT (save money)
./portfolio-nat-manager.sh start primary-nat # Recreate NAT (when needed)
```
